# Pipeline Schemas Reference

Canonical JSON schemas for all pipeline-generated files in `runs/{rootId}/`.  
Generated by the `/pipeline-go` skill. Validated by `npm test` (vitest).  
See `docs/adrs/` for the reasoning behind each schema decision.

---

## Event — `evt_{date}_{NNN}.json`

```json
{
  "schema_version": "0.1",
  "event_id": "evt_{date}_{NNN}",
  "root_id": "{rootId}",
  "story_day": 1,
  "occurred_at": "{date}T12:00:00Z",
  "event_type": "...",
  "canonical_truth": {
    "summary": "...",
    "entities": ["..."],
    "location": "...",
    "concrete_outcome": "..."
  },
  "perception_prompt": {
    "noticeable_surface": "...",
    "ambiguity_level": 0.0
  },
  "tags": ["..."],
  "created_at": "{ISO}",
  "authorial_note": {
    "real_world_inspiration": "...",
    "translation_logic": "...",
    "restraint_reason": "..."
  }
}
```

Key constraints:
- `event_type` is freeform but should be a snake_case label
- `perception_prompt.noticeable_surface` must be something YY can physically encounter — never an announcement (ADR-028)
- `authorial_note.real_world_inspiration` surfaces as `world_anchor` in artifacts (ADR-036)

## Snapshot — `snap_{date}_{branchId}.json`

```json
{
  "schema_version": "0.1",
  "snapshot_id": "snap_{date}_{branchId}",
  "root_id": "{rootId}",
  "branch_id": "{branchId}",
  "story_day": 1,
  "snapshot_date": "{date}",
  "time_policy": "nightly_auto_advance",
  "package_ref": { "package_id": "yysworld-pipeline-v0.1", "package_created_at": "...", "package_hash": "sha256:claude-code-subscription" },
  "model_refs": [{ "provider": "anthropic", "model_id": "claude-code-subscription", "role": "generator", "runtime_class": "interactive" }],
  "event_refs": ["{eventId}"],
  "decision_refs": [],
  "state_before": {
    "condition": { "health": 0.0, "food": 0.0, "attention": 0.0 },
    "inventory": [],
    "active_burdens": [],
    "goals": { "primary": "...", "secondary": "..." },
    "reaction_overrides": {},
    "trait_deviations": {},
    "identity_notes": []
  },
  "state_after": {
    "condition": { "health": 0.0, "food": 0.0, "attention": 0.0 },
    "inventory": [],
    "active_burdens": [],
    "goals": { "primary": "...", "secondary": "..." },
    "reaction_overrides": {},
    "trait_deviations": {},
    "identity_notes": []
  },
  "change_summary": {
    "notable_shift": "...",
    "branch_created": false
  },
  "created_at": "{ISO}"
}
```

Key constraints:
- `condition` must have exactly `health`, `food`, `attention` — never `hunger` (ADR-026)
- All three stats required in every condition object (ADR-033)
- `story_day` and `snapshot_date` are required at top level for the publication gate (ADR-024)
- `state_before` and `state_after` do **not** include `story_day` — early snapshots (days 1–5) contain a spurious `story_day` in `state_before`; this is harmless historical drift, not to be removed (ADR-037)

## Artifact — `art_{date}_{branchId}_summary.json`

```json
{
  "schema_version": "0.1",
  "artifact_id": "art_{date}_{branchId}_summary",
  "artifact_type": "daily_summary",
  "snapshot_id": "snap_{date}_{branchId}",
  "root_id": "{rootId}",
  "branch_id": "{branchId}",
  "package_ref": { ... },
  "model_refs": [{ "provider": "anthropic", "model_id": "claude-code-subscription", "role": "summarizer", "runtime_class": "interactive" }],
  "comparison_contract_version": "0.1",
  "content": {
    "title": "...",
    "tone": "...",
    "narrative": "paragraph one\n\nparagraph two",
    "state_note": "...",
    "summary": "One sentence. Past tense.",
    "state_delta": {
      "food": "0.X → 0.Y",
      "attention": "0.X → 0.Y",
      "health": "0.X → 0.Y"
    },
    "branch_created": false,
    "world_anchor": "..."
  },
  "created_at": "{ISO}"
}
```

Key constraints:
- `state_delta` must include all three stats including health (ADR-033)
- `world_anchor` must be non-empty — copied from `event.authorial_note.real_world_inspiration` (ADR-036)
- Character name in narrative must be `YY` — never a branch label like `alt1`, `alt2-migration` (ADR-032)
- `summary` is one past-tense sentence used as the commit message line

## Decision — `dec_{date}_{NNN}.json`

```json
{
  "schema_version": "0.1",
  "decision_id": "dec_{date}_{NNN}",
  "root_id": "{rootId}",
  "story_day": 1,
  "decision_date": "{date}",
  "decision_status": "evaluated_no_branch",
  "branch_signal": {
    "should_branch": false,
    "confidence": 0.0,
    "reason": "...",
    "focus": null
  },
  "event_refs": ["{eventId}"],
  "snapshot_refs": ["{snapshotId}"],
  "created_at": "{ISO}"
}
```

For executed decisions (branch created), add:
```json
{
  "decision_status": "executed",
  "branch_signal": { "should_branch": true, "confidence": 0.9, "reason": "...", "focus": "..." },
  "branch_created": {
    "branch_id": "branch_{rootId}_{name}",
    "parent_branch_id": "branch_{rootId}_main",
    "fork_condition": "...",
    "fork_day": 1
  }
}
```

Key constraints:
- `branch_signal` is the canonical wrapper (not `signal`, `branch_evaluation`, or `evaluation`) — ADR-037
- `confidence` must be in [0, 1] (ADR-026)
- `decision_status` must be `"evaluated_no_branch"` or `"executed"`
- `dec_2026-04-14_001.json` is an author-created branch decision with a different schema — intentional exception, do not normalize (ADR-037)

## Branch state — `branch_{rootId}_{name}.json`

```json
{
  "schema_version": "0.1",
  "branch_id": "branch_{rootId}_{name}",
  "root_id": "{rootId}",
  "parent_branch_id": null,
  "character_id": "yy",
  "created_at": "{ISO}",
  "status": "active",
  "state": {
    "story_day": 1,
    "condition": { "health": 0.0, "food": 0.0, "attention": 0.0 },
    "inventory": [],
    "active_burdens": [],
    "goals": { "primary": "...", "secondary": "..." },
    "reaction_overrides": {},
    "trait_deviations": {},
    "identity_notes": []
  },
  "drift_flags": [],
  "last_updated_at": "{ISO}"
}
```

Key constraints:
- `state.condition` uses `food` not `hunger` (ADR-026)
- `state.story_day` is the authoritative current day for this branch
- Updated at the end of each pipeline run to match `state_after` from the day's snapshot

## NNN counter rule

`NNN` is zero-padded to 3 digits. For events and decisions, it is the count of all existing files of that type in the directory + 1 at time of generation.

Early decision files (days 1–4) have a NNN collision due to a generator bug — `dec_2026-04-14_001`, `dec_2026-04-15_001`, and `dec_2026-04-16_001` all share `_001`. The IDs remain unique because they include the date. Do not rename these files (ADR-037).
